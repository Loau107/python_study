IP 地址：
IP 地址用于标识网络中唯一一台设备的地址，好比现实生活中的家庭地址
通过 IP 地址可以找到对应设备，然后进行数据通信
查看 IP 地址：
    Windows : ipconfig
    Linux / Mac : ifconfig
测试网络是否正常：
    ping 地址（网络、IP 地址都可以）
本机 IP 地址：
    127.0.0.1


端口：
IP 地址只能帮助数据在网络中找到对应设备，而像微信、QQ等应用程序间的通信，还需要设置一个程序的标识
每个运行的程序都会有一个端口，想要给对应的程序发送数据，就必须找到对应的端口
端口号：
如果把 IP 地址看成学校，那么端口就可以看作教室，而端口号就是教室的门牌号
端口号用于标识设备中唯一的一个端口
端口号的分类：1、知名端口号 2、动态端口号
知名端口号：范围在 0-1023，是已经被不同的知名程序所占有的
动态端口号：范围在 1024-65535，在一个有设置端口的程序运行时，操作系统会随机生成一个动态端口号供程序使用，而程序退出时该端口号会被释放


TCP：
TCP 传输控制协议，是一种面向连接的、可靠的、基于字节流的传输层通信协议
TCP 通信模型相当于生活中的‘打电话’，在通信开始之前，一定要先建立好连接，才能发送数据，通信结束时关闭连接
补充：与其对应的另一种通信模型： UDP 通信模型相当于生活中的‘短信’，不用建立连接就能进行数据通信
特点：1、面向连接 2、可靠连接
可靠连接：1、TCP 是应答机制 2、超时重连 3、错误校验 4、流量控制和阻塞管理


socket：
socket 是进程之间通信的一个工具
负责进程之间的网络数据传输，好比数据的搬运工


生活实例：快递运输
假设有收货信息 xx 省 xx 市 xx 区 xx 楼 xx 号 李先生 188xxxx3456
快递员会先送到你们通过收货地址找到目的地，到达目的地后通过收货人信息去找到对应的收货人，再让收货人确认是否签收，然后将快递交到收货人手里
此实例中：
IP 地址就相当于收货地址
端口就相当于收货人，而端口号就相当于收货人信息
在向收货人确认是否签收时用的是 TCP 通信模型
而 socket 就相当于过程中的快递员
